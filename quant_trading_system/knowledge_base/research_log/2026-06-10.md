### #3 — MACD histogram v3 attempt (minimum-hold variant)

**Built and tested.** Verdict: **REJECT** by addition battery AND **KEEP** by
replacement battery (v3 worse than v1).

**Implementation note on "minimum-hold":** the backtester does not expose
time-in-position to strategies (`StrategyContext.positions` is built from
the simulated long position with `avg_entry_price` but no `entry_date`).
A true minimum-hold rule cannot be implemented without backtester changes
(or per-strategy state persistence, which would be unsafe across runs).
The implementable proxy is to require the exit *signal* to be sustained —
i.e., the histogram-flip exit fires only after `exit_confirmation_bars`
consecutive negative prints (default 3), not on a single negative print.
This addresses the Fri 2026-06-05 META 1-bar round-trip without needing
true position-age tracking.

`equity_momentum_macd_histogram_v3` strategy.md + strategy.py written,
`cli validate-strategy` passed, `cli simulate equity_momentum_macd_histogram_v3
--symbol META` ran cleanly.

**Addition battery (`cli evaluate-add --symbol META`):** REJECT. Failures:
- `num_trades 16 < 20`
- `PSR 0.380 < 0.95`
- `sortino -0.220 < 0.5`
- `OOS/IS sharpe ratio -3.78 < 0.5 (OOS=-1.74, IS=0.46)` — possible overfit

Full-window sharpe -0.23, return -0.68%, max DD -2.82%, 16 trades. The
3-bar confirmation made it WORSE on META, not better. Late exits gave back
gains on the names where the histogram dipped briefly then recovered.

**Replacement battery (`cli evaluate-update equity_momentum_macd_histogram
equity_momentum_macd_histogram_v3 --symbol META`):** KEEP. Sharpe delta
−0.276 (regression, not improvement). Bootstrap p = 0.624. v3 is
statistically indistinguishable from v1 and directionally worse.

**Applied the verdict verbatim. REJECT + KEEP. Did not save.** Cleaned up
the candidate folder via `mcp__cowork__allow_cowork_file_delete` + `rm -rf`.

**Substantive learning:** the META Fri 6/5 1-bar round-trip was a data point
of one, not a systemic weakness of the v1 exit rule. Tightening the exit
threshold actually hurt because it allows momentum-fade trades to keep
running into deeper losses. The signal-quality tradeoff is real: v1's
"hair-trigger" exit IS the risk control. Carry-forward: the
"3-day-min-hold" intuition from operator does not survive backtest. Recommend
not re-attempting v4 without a different mechanism (e.g., a trailing-stop
addition, not a min-hold).
