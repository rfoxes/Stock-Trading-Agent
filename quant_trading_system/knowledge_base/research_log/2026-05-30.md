## Infrastructure failure (the reason this run stops here)

Per task-list step 3, ran the new walk-forward backtester to confirm the
research path is healthy:

```
$ cli simulate equity_trend_following_ema_cross --symbol SPY
{"ok": false, "error": "backtester raised: StrategyContext.__init__()
 missing 1 required positional argument: 'universe'"}

$ cli simulate equity_mean_reversion_bollinger --symbol SPY
{"ok": false, "error": "backtester raised: StrategyContext.__init__()
 missing 1 required positional argument: 'universe'"}
```

Both fail identically. Confirmed downstream impact on the addition battery:

```
$ cli evaluate-add equity_trend_following_ema_cross
{"ok": false, "error": "battery raised: StrategyContext.__init__()
 missing 1 required positional argument: 'universe'"}
```

**Root cause (read-only inspection):** `quant_trading_system/strategy_runtime.py:105`
defines `StrategyContext` with **two** symbol-list fields — `watchlist` (filtered for
the strategy) and `universe` (full composed universe). `universe` is a required
field (no default).

`quant_trading_system/strategy_backtest.py:333` constructs `StrategyContext`
with `watchlist=[symbol]` but **does not pass `universe=...`**. So every
simulate / evaluate-add / evaluate-update / propose-strategy call fails
immediately, before any data is fetched.

The trivial fix is one line in `strategy_backtest.py` (add `universe=[symbol]`
to the constructor call). I did NOT apply it — the manual scopes the
research agent to the strategy library, not the runtime code, and this is
exactly the "infrastructure issue, document and stop" case the operator
manual describes.
